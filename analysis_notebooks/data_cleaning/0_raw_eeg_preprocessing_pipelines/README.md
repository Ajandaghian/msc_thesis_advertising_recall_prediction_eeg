# Raw EEG Preprocessing Pipelines

This folder documents the preprocessing pipelines that were used to transform raw EEG recordings into structured features for downstream analysis.

‼️ Important NDA note:
- The notebooks in this folder were provided by the laboratory as part of the project collaboration.
- Due to NDA and data-sharing restrictions, these notebooks are not distributed for external/public reuse in their original operational form.
- This README provides a methodological description only (objective, expected inputs, expected outputs).

## Pipeline 1: 0_imotions_raw_eeg_preprocessing.ipynb

Objective:
- Parse raw EEG exports and event markers.
- Build segment and bin windows from media/event timing.
- Apply EEG filtering and quality checks.
- Compute band-level spectral features over time windows.

Expected inputs:
- Raw EEG sensor files exported from the acquisition platform.
- Event metadata columns (for example: timestamp, media start/end markers, stimulus/event labels).
- Pipeline parameters such as sampling frequency, frequency bands, artifact thresholds, and bin durations.

Expected outputs:
- Segment-level and bin-level EEG feature table (for example mean PSD or power-spectrum features by band and channel).
- Channel quality table listing invalid channels per source file based on artifact criteria.

## Pipeline 2: 1_preprocessed_eeg_feature_preparation.ipynb

Objective:
- Take preprocessed EEG segment/bin outputs and prepare analysis-ready feature sets.
- Standardize feature naming and structure.
- Perform cleaning and transformations needed before feature engineering/modeling.

Expected inputs:
- Outputs generated by Pipeline 1 (segment/bin EEG feature tables and channel-quality information).
- Configuration for selected features/metrics and any inclusion-exclusion rules.

Expected outputs:
- Curated EEG feature dataset ready for integration with other study tables.
- Intermediate structured tables used in subsequent feature engineering and modeling notebooks.

## What is intentionally not shared

To comply with NDA and privacy constraints, the following are intentionally excluded from public release:
- Raw or processed participant-level EEG datasets.
- Operational notebook versions tied to private laboratory assets.
- Any identifiers, private paths, or metadata that could reveal sponsor or participant information.

## Reproducibility guidance

This folder is intended to describe the pipeline logic and processing stages at a high level. For public-facing reproducibility, use anonymized/synthetic data and re-implement the same sequence of steps in a clean notebook/script that does not depend on restricted assets.

If you need to reproduce the exact processing steps, please contact me to link you with the contact person at the laboratory who can provide access to the original notebooks and data under the appropriate agreements.
